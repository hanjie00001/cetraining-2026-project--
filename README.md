# 个人待办事项管理器 (Personal Todo Manager)

一个简洁实用的命令行待办事项管理工具，使用 Python 编写，支持数据的持久化存储。

## 功能特性

### 核心功能
- ✅ **添加待办事项** — 输入内容、选择优先级（高/中/低）、设置截止日期
- 📋 **查看待办列表** — 支持按优先级或截止日期排序显示
- ✔️ **标记已完成** — 按 ID 标记事项为已完成状态
- 🗑️ **删除待办事项** — 可删除所有已完成事项，或按 ID 单独删除

### 扩展功能
- 🔍 **状态筛选** — 全部 / 未完成 / 已完成 三种视图切换
- 📊 **统计概览** — 显示总数、完成数、完成率等信息
- 💾 **数据持久化** — 自动保存为 `todos.json` 文件，程序重启不丢失

## 快速开始

### 环境要求
- Python 3.6+

### 运行

```bash
python todo_manager.py
```

程序会自动在同一目录下创建 `todos.json` 数据文件。

### 使用示例

```
          ★ 个 人 待 办 事 项 管 理 器 ★

  ┌─ 主菜单 ───────────────────────────┐
  │  1. 添加待办事项                     │
  │  2. 查看待办事项                     │
  │  3. 标记为已完成                     │
  │  4. 删除待办事项                     │
  │  5. 查看统计                         │
  │  0. 退出                             │
  └──────────────────────────────────────┘
```

## 菜单操作说明

| 编号 | 功能 | 说明 |
|------|------|------|
| 1 | 添加待办事项 | 输入内容 → 选择优先级(高/中/低) → 设置截止日期(可选) |
| 2 | 查看待办事项 | 选择排序方式(优先级/截止日期) → 选择筛选状态(全部/未完成/已完成) |
| 3 | 标记为已完成 | 输入待办事项的 ID 完成标记 |
| 4 | 删除待办事项 | 删除所有已完成事项，或按 ID 删除指定事项（含确认） |
| 5 | 查看统计 | 展示待办事项的完成情况统计 |
| 0 | 退出程序 | 安全退出 |

## 数据存储

所有数据以 JSON 格式保存在 `todos.json` 文件中，结构如下：

```json
[
  {
    "id": 1782279806889,
    "content": "完成项目报告",
    "priority": 3,
    "due_date": "2026-06-30",
    "completed": false,
    "created_at": "2026-06-24 10:30:00"
  }
]
```

- `priority`: 3=高, 2=中, 1=低
- `due_date`: 格式 `YYYY-MM-DD`，可为空
- `completed`: `true`=已完成, `false`=未完成

## 涉及知识点

- **数组/列表** — 待办事项的增删改查操作
- **文件读写** — JSON 格式的数据持久化（`json.load` / `json.dump`）
- **排序算法** — 按优先级和截止日期排序（`list.sort`）
- **简单菜单设计** — 命令行交互式菜单循环

## 代码结构

```
todo_manager.py          ← 主程序入口
todos.json               ← 数据文件（自动生成）
```

核心类：

| 类 | 职责 |
|---|---|
| `TodoItem` | 待办事项模型（ID、内容、优先级、截止日期、状态） |
| `TodoManager` | 数据管理（JSON 读写、增删改查、排序筛选） |
| `MenuApp` | CLI 交互界面（主菜单循环、输入输出处理） |

## 注意事项

- 在 Windows 终端（GBK 编码）中运行时，若 emoji 图标显示异常，可执行 `chcp 65001` 切换至 UTF-8 编码后运行
- 添加事项时输入 `Ctrl+C` 可取消当前操作
- 删除操作会要求二次确认，防止误删
